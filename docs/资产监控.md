# 资产监控

## 功能说明

定期监控网站资产变化，检测HTML内容更新，识别新增或修改的资产。

## 执行流程

### 1. 重置网站状态
- 对 `project_{name}_website` 表所有记录的 `status` 字段更新为 `0`（待处理）

### 2. 重放网站，检测变化
- 分批获取待处理网站（status=0）
- 使用 requests 重新请求每个网站
- 将获取的 HTML 导入 `project_{name}_html` 表（通过 `import_html` 方法，MD5去重）
- 对比新旧 `html_len` 字段，判断是否有变化
- 变化阈值：HTML长度差异超过 100 字节视为有变化
- 记录变化的 URL 列表，更新网站状态为 `status=1`（已处理）

### 3. 动态爬取变化的网站
- 对检测到变化的网站调用 `DynamicCrawler` 进行动态爬取
- **注意**：请求数量需小于等于 `browser_thread` 配置值，避免浏览器队列溢出
- 获取页面动态数据：
  - `title` - 页面标题（可能被JS动态修改）
  - `screenshot` - 页面截图
  - `html_browser_md5` - 浏览器渲染后的HTML的MD5
- 更新 `time_update` 字段为当前时间

### 4. 更新数据库
更新变化网站的以下字段：
```json
{
  "title": "新标题",
  "screenshot": "截图文件名",
  "html_browser_md5": "浏览器渲染HTML的MD5",
  "time_update": "2026-04-12 11:30:00"
}
```

## 数据表关系

```
project_{name}_website (站点表)
├── status: 0=待处理, 1=已处理
├── html_len: 静态HTML长度
├── html_browser_md5: 浏览器渲染HTML的MD5
├── title: 页面标题
├── screenshot: 截图文件名
└── time_update: 更新时间（变化时更新）

project_{name}_html (HTML表)
├── html: HTML内容
├── html_md5: MD5值（去重用）
├── html_len: HTML长度
└── status: 0=未处理, 1=已处理
```

## 使用方式

### 命令行测试
```python
from service.spider.asset_monitor import AssetMonitor

monitor = AssetMonitor('Tencent')
result = monitor.monitor()
print(result)
```

### API调用
```python
POST /api/asset_monitor/start
{
    "project_name": "Tencent"
}
```

## 配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `browser_thread` | 浏览器并发数 | 10 |
| `http_thread` | HTTP请求并发数 | 10 |
| `timeout` | HTTP请求超时时间（秒） | 8 |
| `change_threshold` | HTML长度变化阈值（字节） | 100 |

## 注意事项

1. **服务依赖**：动态爬取需要 Chrome CDP 和 Mitmproxy 服务运行中
2. **并发控制**：变化的网站数量不应超过 `browser_thread`，否则会分批处理
3. **时间更新**：只有检测到变化的网站才会更新 `time_update` 字段
4. **MD5去重**：HTML表使用 `html_md5` 去重，相同内容不会重复存储
