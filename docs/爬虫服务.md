# 爬虫服务

## 概述

爬虫服务是系统的被动扫描模块，通过 Mitmproxy 代理捕获流量，并对流量进行分析处理，收集子域名、网站、HTTP请求等资产数据。

**核心组件：**
- `SpiderCore` - 爬虫核心调度器
- `AssetMonitor` - 资产监控器

**服务开关：** `spider_service=1`

## 服务流程

### 1. SpiderCore 运行流程

```
while is_running:
    1. 检查 spider_service == 1
    2. 检查是否有待处理数据 (traffic/html/website)
    3. 条件满足则执行处理流程，否则睡眠10秒
    
处理流程:
    ├── 获取未处理流量 (limit=200)
    ├── 流量过滤分类
    │   ├── 子域名收集
    │   ├── 网站收集
    │   └── HTTP请求收集
    ├── 动态爬虫 (自动从数据库获取URL)
    └── HTML收集
```

### 2. AssetMonitor 运行流程

```
while is_running:
    1. 检查 monitor_service == 1
    2. 执行监控流程，否则睡眠10秒
    
监控流程:
    ├── 重置所有网站状态 (status=0)
    ├── 重放网站，检测HTML变化
    │   └── 变化阈值: >20%
    └── 动态爬取变化的网站
        └── 更新: title, screenshot, html_browser_md5, time_update
    
    完成后切换:
    └── monitor_service=0, spider_service=1
```

## 核心模块

### 1. SpiderCore (core.py)

爬虫核心调度器，协调各子模块工作。

**主要方法：**

| 方法 | 说明 |
|------|------|
| `__init__` | 初始化，加载各子模块 |
| `check_pending_data` | 检查是否有待处理数据 |
| `fetch_unprocessed_traffic` | 获取未处理的流量数据 |
| `run_dynamic_crawler` | 运行动态爬虫 |
| `run` | 运行爬虫主流程（持续循环） |

### 2. AssetMonitor (asset_monitor.py)

资产监控器，检测网站变化。

**主要方法：**

| 方法 | 说明 |
|------|------|
| `reset_websites_status` | 重置所有网站状态为待处理 |
| `replay_websites` | 重放网站，检测HTML变化 |
| `crawl_changed_websites` | 动态爬取变化的网站 |
| `monitor` | 执行完整监控流程 |
| `run` | 运行监控服务（持续循环） |

## 子模块

### 1. SubdomainCollector (subdomain.py)
子域名收集器，从流量中提取和保存子域名。

### 2. WebsiteCollector (website.py)
网站收集器，收集网站基本信息（URL、标题、状态码等）。

### 3. HTTPCollector (http.py)
HTTP请求响应收集器，收集 GET 请求的响应内容。

### 4. HTMLCollector (html.py)
HTML大文件收集器，存储大型HTML内容。

### 5. DynamicCrawler (dynamic_crawler.py)
动态爬虫，通过 Chrome CDP 接口进行动态页面爬取。
- 支持 JavaScript 渲染
- 自动获取页面截图
- 提取页面标题

### 6. BrowserCDP (browser_cdp.py)
Chrome CDP 协议控制模块，封装 Chrome DevTools Protocol 操作。

### 7. HTTPStandardization (http_standardization.py)
HTTP标准化处理模块。

## 数据库表

| 表名 | 说明 | 状态字段 |
|------|------|----------|
| `project_{name}_traffic` | HTTP流量 | status: 0=未处理, 1=已处理, scaner_status: 0=未扫描 |
| `project_{name}_website` | 网站信息 | status: 0=未处理, 1=已处理 |
| `project_{name}_subdomain` | 子域名 | - |
| `project_{name}_html` | HTML大文件 | status: 0=未处理, 1=已处理 |

## 服务开关

通过 `service_lock` 配置控制服务开关：

| 服务 | 配置项 | 说明 |
|------|--------|------|
| 爬虫服务 | `spider_service=1` | 开启流量处理和资产收集 |
| 资产监控 | `monitor_service=1` | 开启网站变化监控 |
| 自动扫描 | `scaner_service=1` | 开启漏洞扫描 |

**服务切换逻辑：**
```
AssetMonitor 完成后:
    monitor_service = 0
    spider_service = 1
```

## 注意事项

1. **服务独立性**：爬虫服务和资产监控服务可以同时运行，但由 `service_lock` 控制开关
2. **循环检测**：服务会持续循环检测，每 10 秒检查一次开关状态
3. **Chrome 依赖**：动态爬虫依赖 Chrome headless 服务
4. **批量处理**：使用 `http_thread` 和 `browser_thread` 控制并发数
5. **变化检测**：HTML 变化阈值默认为 20%
