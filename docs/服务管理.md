# 服务管理

## 概述

服务管理模块负责管理系统运行时所需的各项服务，包括浏览器代理服务、漏洞扫描服务和资产监控服务。

**设计原则：**
- 独立于 Flask 运行，修改 Flask 代码不会影响服务
- 通过 API 提供服务状态监测（端口 5002）
- 支持 Chrome 自动重启和状态监控

## 服务列表

### 1. Mitmproxy（流量捕获）
- **端口**: 18081
- **启动方式**: start.bat 独立启动
- **功能**: 捕获 HTTP/HTTPS 流量，写入 MongoDB
- **配置文件**: `config.json` 中的 `mitmproxy_port`

### 2. Chrome Headless（无头浏览器）
- **CDP 端口**: 19227
- **代理**: mitmproxy (18081)
- **功能**: 被动爬虫、资产抓取
- **用户数据目录**: `project_data/{项目名}/chrome_data/headless`
- **配置文件**: `config.json` 中的 `chrome_path`, `chrome_cdp_port`

### 3. Chrome Normal（普通浏览器）
- **CDP 端口**: 19228
- **代理**: Burp (8080) 或 mitmproxy (18081)
- **功能**: 爬虫、正常浏览
- **用户数据目录**: `project_data/{项目名}/chrome_data/normal`
- **配置文件**: `config.json` 中的 `chrome_path`, `chrome_spider_cdp_port`

### 4. Burp Suite（可选）
- **端口**: 8080
- **功能**: HTTP 代理、流量拦截
- **启动条件**: `burp_path` 和 `burp_port` 都配置了才启动
- **配置文件**: `config.json` 中的 `burp_path`, `burp_port`

### 5. 自动扫描服务
- **功能**: 漏洞自动扫描
- **依赖**: `VulnerabilityScanner` 类
- **特性**:
  - 参数去重 (`enable_dedup=True`)
  - 批量处理 (`batch_size=10`)
  - 服务开关: `scaner_service=1`

### 6. 资产监控服务
- **功能**: 资产变化监控
- **依赖**: `AssetMonitor` 类
- **特性**:
  - HTML 变化检测
  - 动态爬取更新
  - 服务开关: `monitor_service=1`

## 启动顺序

```
1. 等待项目运行
   └── callback_project_config() 返回有效项目

2. 启动 Chrome (headless)
   └── CDP: 19227, 代理: mitmproxy (18081)

3. 启动 Chrome (normal)
   └── CDP: 19228, 代理: Burp (8080) 或 mitmproxy (18081)

4. 启动 Burp Suite (如果配置了)
   └── burp_path 和 burp_port 都非空

5. 启动自动扫描服务
   └── VulnerabilityScanner.auto_scan()

6. 启动资产监控服务
   └── AssetMonitor.run()
```

---

## 停止顺序

```
1. 停止资产监控服务
2. 停止自动扫描服务
3. 停止 Burp Suite
4. 停止 Chrome (normal)
5. 停止 Chrome (headless)
```

## 配置文件

`config.json` 中的相关配置项：

```json
{
    "chrome_path": "C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe",
    "burp_path": "C:\\BurpSuitePro",
    "burp_port": 8080,
    "chrome_cdp_port": 19227,
    "chrome_spider_cdp_port": 19228,
    "mitmproxy_port": 18081
}
```

## API 接口

状态监测 API 端口: **5002**

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/service/status` | GET | 获取所有服务状态 |
| `/api/service/start` | GET | 启动所有服务 |
| `/api/service/stop` | GET | 停止所有服务 |
| `/api/service/restart` | GET | 重启所有服务 |

**响应格式：**
```json
{
    "mitmproxy": {"running": true, "port": 18081},
    "chrome_headless": {"running": true, "port": 19227},
    "chrome_normal": {"running": true, "port": 19228},
    "burp": {"running": true},
    "auto_scan": {"running": true},
    "asset_monitor": {"running": true}
}
```

## 服务状态开关

服务通过 `service_lock` 配置控制开关：

| 服务 | 配置项 | 说明 |
|------|--------|------|
| 自动扫描 | `scaner_service=1` | 开启自动扫描 |
| 资产监控 | `monitor_service=1` | 开启资产监控 |
| 爬虫服务 | `spider_service=1` | 开启爬虫服务 |

## 注意事项

1. **Mitmproxy 独立启动**: Mitmproxy 由 `start.bat` 独立启动，不通过 `service_manager.py`
2. **Burp 可选**: 如果不配置 `burp_path` 和 `burp_port`，Burp 不会启动，Chrome normal 使用 mitmproxy 作为代理
3. **Chrome 状态监控**: 每 10 秒检测一次 Chrome 端口，异常时自动重启
4. **线程守护**: 所有后台服务线程都设置为 `daemon=True`，主程序退出时自动终止
