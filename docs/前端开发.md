## 开发标准

### 1. 前后端分离
- Flask 遵循前后端分离原则，所有数据交互通过 API 进行
- 不允许使用 Flask 模板数据传输，必须通过 JavaScript API 调用
- 前端采用单页应用(SPA)架构

### 2. 前端架构
- HTML5 + CSS3 自适应布局
- 通过模块化的 JavaScript 文件实现各个功能模块
- 每个功能模块独立维护 JS 和 CSS 文件

### 3. 导航管理
- 左侧菜单栏与右侧功能显示区域
- 功能显示采用 Tab 管理机制
- Tab 与浏览器导航 Hash 对应，支持前进/后退

### 4. CSS 命名规范
- 使用特定前缀避免样式冲突（如 `facai-`、`spider-`、`scaner-`）
- 限定作用域，避免全局样式污染
- 使用 `!important` 覆盖第三方库的样式冲突

### 1. core.js
- 应用程序入口
- 初始化 TabManager 和 HashManager
- 绑定菜单事件

### 3. hash.js
- Hash 导航管理
- 监听浏览器 Hash 变化

### 4. pageup.js
- 分页组件
- 支持总页数、上下翻页、页码跳转
- 可定制化的分页样式

### 5. modal.js
- 模态框管理组件
- 统一的模态框创建和管理接口

## 模块开发规范

### 1. 模块结构
每个功能模块应遵循以下结构：
```javascript
function ModuleName() {
    // 模块状态
    this.container = null;
    
    // 渲染方法（由 TabManager 调用）
    this.render = function(data, container) {
        this.container = container;
        // 渲染 HTML
        // 加载数据
        // 绑定事件
    };
    
    // 其他方法
    this.loadData = function() { };
    this.bindEvents = function() { };
}
```

### 2. CSS 样式隔离
- 使用唯一前缀（如 `facai-`、`spider-`、`scaner-`）
- 限定在容器内（如 `.spider-container .card`）
- 使用 `!important` 覆盖全局样式

### 3. API 调用
- 统一使用 `api/` 目录下的 API 模块
- 使用 jQuery.ajax 进行请求
- 统一错误处理

## 样式冲突解决方案

### 问题描述
`head.css` 定义了全局样式，会覆盖模块中使用的 Bootstrap 等第三方库样式。

### 解决方案
1. **使用特定前缀**：为模块类名添加唯一前缀（如 `facai-`、`spider-`、`scaner-`）
2. **限定作用域**：使用 `.module-container .component` 选择器
3. **强制覆盖**：对关键样式使用 `!important`
4. **恢复默认**：为被覆盖的 Bootstrap 样式重新定义

### 示例
```css
/* 恢复 spider 模块中的 Bootstrap 按钮样式 */
.spider-container .btn {
    padding: .375rem .75rem !important;
    border-radius: .25rem !important;
    /* 覆盖全局自定义样式 */
}
```

## 注意事项

1. **新增模块**：需在 `index.html` 中添加菜单项和脚本引用
2. **脚本加载顺序**：API 脚本需在模块脚本之前加载
3. **样式隔离**：每个模块应有独立的 CSS 文件，避免全局污染
4. **代码规范**：遵循模块化开发规范，保持代码清晰可维护
